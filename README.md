# Конструктор «Детали» — приложение-сетка

Веб-приложение для планирования мероприятий студии «Детали».
Параллельно с camp-generator и intensive-generator.

**Что делает:**
- Менеджеры планируют события в календарной сетке месяца
- Справочники форматов, педагогов и шаблонов событий ведутся внутри приложения
- ИИ-помощник пишет описания мероприятий
- Опубликованные события улетают в Google Sheets «События» и далее в Альфа CRM
- Несколько менеджеров видят, кто сейчас редактирует
- Защищено паролем

**Архитектура:**
- `index.html` — главное приложение (фронтенд)
- `view.html` — публичный просмотр опубликованных расписаний (без пароля)
- `Code.gs` — серверная часть (Google Apps Script): хранение данных, ИИ-прокси, мост в лист «События»
- `alfa_integration.gs` — мост из листа «События» в Альфа CRM (живёт отдельно в той же таблице)

## Установка — пошагово

### 1. Подготовка Google Sheets

1. Откройте таблицу **«Детали — Сетка планирования»** (или создайте новую, если ещё не залили xlsx)
2. Скопируйте **ID таблицы** из URL — это часть между `/d/` и `/edit`:
   `https://docs.google.com/spreadsheets/d/`**`1AbCDeFgHi...XyZ`**`/edit`

### 2. Развёртывание Apps Script

1. В этой таблице: **Расширения → Apps Script**
2. Если уже есть `alfa_integration.gs` — оставьте его как **отдельный файл** в проекте, он не конфликтует
3. Создайте **новый файл** `Code.gs` (или замените тот, что был для camp/intensive — этот код совместим со всеми тремя проектами)
4. Вставьте содержимое `Code.gs` из этого репозитория
5. **Подставьте `SHEET_ID`** в самом верху файла:
   ```js
   const SHEET_ID = '1AbCDeFgHi...XyZ';
   ```
6. Сохраните (Ctrl+S)

### 3. Создайте служебные листы (один раз)

В редакторе Apps Script:
1. Сверху, в выпадающем списке функций, выберите `_testInitDetails`
2. Нажмите **«Выполнить»**
3. (Опционально) Запустите `_testSeedDetails` — заполнит справочники начальными данными (16 форматов + 14 педагогов)
4. Откройте таблицу — увидите новые листы:
   - `Форматы_детали`
   - `Педагоги_детали`
   - `Шаблоны_детали`
   - `Опубликованные_детали`
   - `Черновики_детали`
   - `Сессии_детали`

### 4. Настройте API-ключ Anthropic для ИИ

1. В редакторе Apps Script: **⚙ Project Settings → Script Properties → Add property**
2. Имя: `ANTHROPIC_API_KEY`
3. Значение: ваш ключ (начинается с `sk-ant-...`)
4. Сохранить

Без этого шага кнопка «✨ Сгенерировать ИИ» не будет работать.

### 5. Опубликуйте Apps Script как Web App

1. В редакторе Apps Script: **Deploy → New deployment**
2. Выберите тип: **Web app**
3. Description: `Детали v1`
4. Execute as: **Me** (ваш аккаунт)
5. Who has access: **Anyone** ⚠ (это нужно, чтобы фронтенд с GitHub Pages мог дёргать API; данные защищены паролем на стороне фронта)
6. Нажмите **Deploy**
7. Скопируйте **Web app URL** — это длинная ссылка вида `https://script.google.com/macros/s/AKfyc.../exec`

### 6. Настройте фронтенд

В файлах `index.html` и `view.html` найдите строку:
```js
APPS_SCRIPT_URL: 'PUT_YOUR_APPS_SCRIPT_WEB_APP_URL_HERE'
```
и подставьте URL, полученный на шаге 5.

В `index.html` пароль уже стоит:
```js
PASSWORD_HASH: '19082026'
```
Если хотите сменить — поменяйте в этой строке.

### 7. Залейте на GitHub Pages

1. Создайте новый репозиторий `details-generator` в аккаунте Jane3dMp
2. Загрузите туда три файла: `index.html`, `view.html`, `Code.gs`, `README.md`
3. **Settings → Pages → Source: Deploy from a branch → main → / (root) → Save**
4. Через 1-2 минуты приложение будет доступно по адресу:
   `https://jane3dmp.github.io/details-generator/`
5. Публичная страница расписания: `https://jane3dmp.github.io/details-generator/view.html`

## Как работает связь между всеми частями

```
[ Менеджер ] → [ index.html (GitHub Pages) ] → [ Code.gs Web App ] → [ Google Sheets ]
                                                      ↓                       ↓
                                              [ Anthropic API ]         [ лист «События» ]
                                              (ИИ-генерация)                   ↓
                                                        ↓             [ Alfa.gs (та же среда) ]
                                                        └──────────→         ↓
                                                                  [ Альфа CRM API ]
```

**Полный цикл одной кнопкой:** менеджер жмёт «🚀 Опубликовать в Альфу», и `Code.gs`:
1. Сохраняет снимок месяца в `Опубликованные_детали`
2. Синхронизирует все события в лист «События» (для истории и контроля)
3. **Сразу же** для событий с галочкой «Готово в Альфу»: логинится в Альфу, создаёт группы и уроки, записывает ID в лист «События»
4. Возвращает менеджеру окно с результатом: «Отправлено: 5, ошибок: 0»

Менеджер **не открывает Google Sheets вообще**. Лист «События» работает как полка истории — туда можно зайти проверить, что записалось, увидеть ID групп, отловить ошибки. Старое меню «🎟️ Альфа CRM → Отправить помеченные» в таблице **остаётся** как ручной режим — на случай, если что-то надо переотправить точечно.

**Зависимость:** `Code.gs` использует функции `alfaLogin_`, `alfaCall_`, `extractCreatedId_`, `formatTime_`, `addHours_`, `toInt_` из `Alfa.gs`. Поэтому в проекте Apps Script должны лежать **оба файла**.

## Совместное редактирование

Каждый менеджер при входе вводит своё имя. Раз в 30 секунд клиент шлёт heartbeat. Если другой менеджер открыл тот же месяц, в шапке появится:

> ⚠ Сейчас редактирует: Аня

Это **предупреждение, а не блокировка** — в Google Sheets всё равно работает «последняя запись побеждает». Если важно избежать конфликтов, договоритесь, кто на каком месяце работает.

## Что положить в справочники

После запуска `_testSeedDetails` справочники наполнятся. Затем менеджеры:

- **Форматы** (`Форматы_детали`) — выпадающий список в редакторе события. Поля: название, категория, шаблон описания, цена/лимит/длительность по умолчанию.
- **Педагоги** (`Педагоги_детали`) — выпадающий список в редакторе. Поле «ID в Альфе» — для будущей привязки педагога к создаваемой группе (пока не используется, оставлено на потом).
- **Шаблоны** (`Шаблоны_детали`) — заполняются автоматически, когда менеджер сохраняет событие как шаблон. Используются для быстрого создания «такого же, но в другой день».

Прямое редактирование справочников — в самой Google Sheets. Через приложение пока можно только **добавлять** новые форматы и педагогов, но не удалять (это сделано специально — чтобы не сломать существующие события). Если нужно удалить — заходите в Google Sheets и проставляете в колонке «Активен» значение «Нет».

## Безопасность

- **Пароль** клиентский — `19082026`, проверяется в `index.html`. Это не суперзащита, но достаточно, чтобы случайный человек не зашёл. Для смены — поменяйте `PASSWORD_HASH` в `index.html`.
- **API Альфы** живёт только в `alfa_integration.gs` (внутри Google Apps Script). На фронтенд не утекает.
- **Anthropic API ключ** живёт в Script Properties. На фронтенд не утекает — там только запрос к Apps Script.
- **Web App открыт для anyone** — но он отвечает только на известные действия (`publish`, `saveDraft`, `claude` и т.д.). Если беспокоит — можно ограничить через Cloudflare/Google IAP.

## Когда что-то сломалось

- **«APPS_SCRIPT_URL не настроен»** — значит вы не подставили URL в `index.html`. См. шаг 6.
- **Тосты «Не удалось связаться с сервером»** — проверьте, что Web App опубликован с `Anyone` access. Иногда после правок Code.gs нужно сделать `Deploy → Manage deployments → ✏ → New version → Deploy` заново.
- **«ANTHROPIC_API_KEY не настроен»** — см. шаг 4.
- **События не появились в листе «События»** — запустите вручную: в Apps Script функцию `detailsPingEvents_` (через тест/логи) и проверьте, что лист с именно таким названием существует.
- **«Сейчас редактирует: пусто»** — у того менеджера не было ввода имени или сессия протухла (5 мин неактивности).

## Что не сделано в этой версии (план на v2)

- Drag-and-drop переноса событий между датами
- Превью афиши/постера события
- Печать расписания на месяц одним документом
- Прямая отправка в Альфу из конструктора (без промежуточного шага в таблице)
- Подтягивание актуального ID педагога/предмета из Альфы при создании группы
